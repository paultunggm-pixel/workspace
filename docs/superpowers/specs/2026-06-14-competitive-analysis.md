# Claude Code Desktop vs Claude Code Studio — 竞品对比分析

> 日期：2026-06-14 | 背景：Anthropic 于 2026 年发布 Claude Code Desktop 正式版


## Claude Code Desktop 核心能力

| 功能 | 说明 |
|------|------|
| **Chat / Cowork / Code** | 三模式合一：对话、知识工作代理、编程 |
| **Claude Preview** | 内嵌浏览器，代码改后自动刷新渲染，点击页面元素即可发指令 |
| **并行会话** | 每会话独立 Git Worktree，互不干扰 |
| **内嵌终端** | 在应用内跑 build/test，不用切窗口 |
| **拖拽布局** | 终端/预览/差异/对话面板自由拖拽成自定义网格 |
| **PR 监控** | 跟踪 CI 状态，自动修复失败，自动合并 |
| **Computer Use** | 控制浏览器/鼠标/键盘完成任务 |
| **Routines** | 云端定时/触发自动化，无需本地会话在线 |
| **会话可移植** | CLI → Desktop → Web → Mobile 无缝切换 |
| **侧聊 (⌘+;)** | 任务中途分叉提问，不打断主流程 |
| **5 级权限** | 询问/自动编辑/计划/自动/绕过 |


## 对比矩阵

| 维度 | Claude Code Desktop | Claude Code Studio | 差异判断 |
|------|------|------|------|
| **AI 模型** | 仅 Claude (Anthropic) | DeepSeek/Qwen/Anthropic/Gemini/OpenAI 多选 | ✅ Studio 更灵活 |
| **成本可见性** | 订阅制 ($20-200/月) | 按量计费，余额+Token 实时可见 | ✅ Studio 更透明 |
| **项目管理** | 无分类体系 | 分类→项目两级，项目概览 .md | ✅ Studio 独有 |
| **发布会话**| 单元能力 | 三类发布(Web/移动/桌面)·构建→分发·本地安装 | ✅ Studio 更完整 |
| **服务集成** | MCP 协议通用支持 | 30+ 服务目录 + GitHub 深度集成 | ⚖️ 各有侧重 |
| **CLAUDE.md 管理** | 文件编辑 | 可视化表单 + 开关 + 标签 | ✅ Studio 更人性化 |
| **知识沉淀** | 无 | 沉淀到 CLAUDE.md / 项目 Skills 快捷按钮 | ✅ Studio 独有 |
| **Claude Preview** | ✅ 内嵌浏览器 + 点击元素指 | ⚠️ 预览 Tab 仅手动刷新 + 浏览器打开 | 🔴 差距 |
| **内嵌终端** | ✅ 应用内跑命令 | ❌ 无 | 🔴 差距 |
| **Computer Use** | ✅ 控制桌面 | ❌ 未考虑 | 🟡 差距（研究预览阶段） |
| **并行会话** | ✅ 多会话+Git Worktree | 单会话（有意设计选择） | ⚖️ 设计理念不同 |
| **拖拽布局** | ✅ 自定义网格 | 固定布局 | 🟡 次要 |
| **Routines** | ✅ 云端定时任务 | ❌ 未考虑 | 🟡 次要 |
| **会话可移植** | ✅ 跨设备 | ❌ 未考虑 | 🟡 次要 |


## 建议补上的短板

### 🔴 必须对齐

**1. Claude Preview（内嵌浏览器）**

当前预览 Tab 只能手动刷新或浏览器打开。建议升级为：

```
🎨 预览
┌─ 内嵌 WebView ──────────────────────────────┐
│  实时渲染 HTML · 代码改动后自动热刷新         │
│  点击页面元素 → 弹出指令输入 → Claude 修改代码 │
└──────────────────────────────────────────────┘
[📱 375px] [💻 1440px] [🔗 浏览器] [🔄 刷新]
```

技术可行性：macOS WKWebView 原生支持，可注入 JS 实现元素选择+指令发送。

**2. 内嵌终端**

当前完全依赖外部终端。建议在中央区增加一个可折叠的底部终端面板：

```
┌─ 终端 ─────────────────────────────────────┐
│ $ npm run build                              │
│ > claude-code-studio@1.0.0 build            │
│ > swift build                                │
│ Build complete! (12.3s)                      │
└──────────────────────────────────────────────┘
```

快捷键 `⌘\`` 切换显示。技术可行性：macOS 原生 Terminal 组件或伪终端 (PTY)。

### 🟡 考虑增加

**3. 侧聊 (⌘+;)**

当前只能主对话区交流。增加一个小型侧聊面板，用于任务中途快速提问分叉而不打断主流程。

### ⚪ 暂不跟进

| 功能 | 原因 |
|------|------|
| Computer Use | 仍处研究预览，稳定性不足 |
| Routines | 依赖云端基础设施，个人产品暂不需要 |
| 拖拽布局 | 固定布局对目标用户更友好 |
| 会话可移植 | 本地单机使用场景，暂不跨设备 |
| 多会话+Worktree | 我们做了深思熟虑的单会话设计，沉淀+压缩替代多会话 |
| PR 监控 | GitHub Actions 已覆盖基本需求 |


## 差异化定位总结

```
Claude Code Desktop = 全能 IDE 替代品（工程师向）
Claude Code Studio  = 产品创造工作台（产品经理/全栈向）

Desktop 更强：   预览、终端、并行、Computer Use
Studio 更强：    多模型、项目体系、发布流程、成本透明、知识沉淀
```

**核心差异化价值：** Studio 不只是一个"更好的 Claude Code 客户端"，而是一个从**构思→设计→开发→发布→沉淀**的完整产品创造流水线。Desktop 是更强大的 IDE，Studio 是更完整的工坊。
