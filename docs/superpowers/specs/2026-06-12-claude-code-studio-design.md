## 1. 产品概述

### 1.1 定位

Claude Code Studio 是一款 macOS 原生桌面应用，作为 Claude Code 的外壳程序，将 AI Agent 能力**产品化、可视化、易用化**。设计理念借鉴 Codex 客户端——以需求/设计人员为本，降低工程师知识门槛。

### 1.2 核心目标

- **信息聚合**：将分散在终端、VS Code、浏览器中的信息集中到一个窗口
- **项目管理**：以项目为中心（而非会话），建立分类体系
- **可视化预览**：方案/设计/代码/部署一站预览
- **模型管理**：完全内建的模型配置，零外部依赖
- **酷炫体验**：SwiftUI 原生，流畅动画，现代设计

### 1.3 技术选型

| 项 | 选择 |
|------|------|
| 框架 | SwiftUI (macOS 13+) |
| 语言 | Swift |
| 密钥存储 | macOS Keychain (`SecItem` API) |
| 配置存储 | `~/.claude-code-studio/` (自有 JSON 格式) |
| 与 cc 兼容 | 双向同步 `~/.claude/settings.json` |

## 3. 左侧栏 — 模型引擎区

### 3.1 主模型下拉选择器

- 展开后按**提供商分组**（DeepSeek / Anthropic / OpenAI 兼容）
- 当前选中项标 ✓ + 蓝色高亮
- 底部「+ 添加模型提供商...」入口
- 支持键盘导航（↑↓ Enter）

### 3.2 三键快切

- 三个预设模型快捷按钮：⚡ Flash / 🧠 Pro / 🔮 Opus
- 当前激活按钮蓝色边框高亮
- 切换即时生效，影响后续对话

### 3.3 余额 / 用量卡片

- 显示当前模型提供商的余额（大字体金额 + 颜色：绿 >50%、黄 10-50%、红 <10%）
- 月度用量进度条
- 今日调用次数 + Token 消耗
- 底部链接到 provider 用量页面（如 `platform.deepseek.com/usage`）
- Token 过期后显示「⚠️ 授权已过期」提示重新授权

### 3.4 完全内建的模型设置

#### 设计原则

- **零依赖**：不需要用户安装 `cc switch`
- **自有配置**：`~/.claude-code-studio/providers.json`
- **cc 兼容**：首次启动自动导入 `~/.claude/settings.json` 已有配置，每次修改双向同步
- **密钥安全**：100% macOS Keychain，不落盘

#### 添加提供商流程

```
步骤① 选择提供商 → 步骤② 输入 API Key → 步骤③ 验证 & 启用
```

- 步骤①：预设列表（DeepSeek / Anthropic / OpenAI 兼容）+ 自定义端点
- 步骤②：粘贴 Key，立即通过 `SecItemAdd()` 入 Keychain，输入框显示 `sk-●●●●●●●●d8f2`
- 步骤③：自动调用 provider 的 models list API 测试连接，拉取余额

#### 模型设置面板（⚙️ 管理入口）

- 每个提供商卡片：名称 · 连接状态 · API Key 摘要 · 默认模型选择
- 展开后：Key 管理（👁️显示 / 删除）+ 可用模型列表 + 余额信息
- 「+ 添加」按钮 → 触发三步引导

### 3.5 首次初始化引导

- 检测到 `providers.json` 为空 → 模型选择器位置显示引导卡片
- 引导内容：「🧠 尚未配置模型」→ 点击「+ 添加提供商」→ 进入三步设置
- 不使用 `cc switch`，完全在 App 内完成

### 3.6 数据流

```
~/.claude/settings.json  ←→  App 双向同步 (当前模型选择)
~/.claude-code-studio/   ←   App 自有配置 (providers + preferences)
macOS Keychain           ←   API Key (仅 App 可读，不落盘)
provider API             →   App 定时拉取 (余额, 每小时刷新)
```

### 4.1 数据结构

### 4.2 分类操作

| 操作 | 行为 |
|------|------|
| 新建分类 | 点击 `+📂` → 输入名称 → 创建空分类 |
| 删除分类 | 仅当分类内无项目时可删，否则提示「请先移走项目」 |
| 展开/收起 | 点击分类名切换 |

### 4.3 项目操作

| 操作 | 行为 |
|------|------|
| 新建项目 | 点击 `+📁` → 输入项目名 → 选择所属分类 |
| 移动项目 | 右键 →「移动到...」→ 选择目标分类 / 新建分类 |

| 重命名 | 双击项目名编辑 |

### 4.4 删除项目三次确认

1. **第1次**：输入项目名称确认
2. **第2次**：显示警告「包含 N 个会话，删除后不可恢复」
3. **第3次**：5 秒倒计时确认按钮

### 4.5 会话操作

- 展开项目 → 显示会话列表，按时间分组（今天 / 昨天 / 更早）
- 点击会话 → 中央区加载对话历史
- 新建会话：「+ 新建会话」按钮 → 中央区空白对话页
- 会话状态：●活跃 / ◐暂停 / ○完成

## 8. 安全设计

### 8.1 密钥管理铁律

| 原则 | 实现 |
|------|------|
| 🚫 不存明文 | API Key 绝不落盘（plist / UserDefaults / JSON） |
| 🔐 Keychain | 仅通过 `SecItemAdd/Delete/Copy` 操作 macOS 钥匙串 |
| 🔗 OAuth | 用量查询通过系统 Safari 完成 OAuth，App 只拿 auth code |
| 🗑️ 登出即毁 | `SecItemDelete` 彻底清除，无残留 |
| 👁️ 掩码显示 | UI 中仅显示 `sk-●●●●●●●●d8f2`（后4位） |

### 8.2 第三方登录

- 用量数据（如 DeepSeek 余额）通过 OAuth 2.0 获取
- 登录完全在 `platform.deepseek.com` 完成（系统 Safari）
- App 不嵌入 WebView，无法捕获用户在第三方网站的任何输入
- 回调仅返回一次性 auth code（5分钟过期）

## 11. CLAUDE.md 全局设置

### 11.1 设计原则

- **人性化编辑**：标签点选代替手写 yaml，开关代替 true/false，自然语言文本框代替严格 schema
- **保留原始入口**：每个设置面板提供「查看原始 MD」按钮
- **Apple 风格排版**：iOS Settings 式分组列表，白色 group-body + 灰色背景分组

### 11.2 设置板块

#### 交流偏好
- **工作语言**：标签单选（简体中文 / 繁體中文 / English / + 其他）
  - 点击「+ 其他」→ 内联输入框弹出 → 确认后加入标签列表
- **回复风格**：标签单选（简洁 / 适度 / 详尽）

#### 权限与安全（合并板块，含三个子板块）

**询问频率**（radio 单选，描述右置）：
- 关键确认（默认）：隐私、安全、文件删除等敏感操作询问，常规自主
- 每步询问：所有操作执行前确认
- 高度自主：AI 全权执行，可随时暂停或回滚

**读写权限**（radio 单选，二选一）：
- 指定授权读写单一目录（默认）：Claude 仅能读写指定文件夹
  - 选中后显示路径输入行 + 📂 选择按钮，默认 `~/Documents/Claude/`
- 全局授权读写本地文件数据：Claude 可访问用户目录下任意文件

**安全策略**（标签，多选）：
- 不存储密钥 / 外发前确认

#### 编码风格（开关列表，副标题右侧并排）
- 动手前先对齐需求
- 简单优先
- 精准修改
- 修改后自动代码审查

#### 自定义规则（文本框，min-height: 144px）
- 自然语言补充全局规则

### 11.3 排版规范

- 一级标题 `.group-header`（11px W600 灰色）作为板块标题
- 二级标题 `.row-label`（13px）作为字段名，`.row-hint`（11px 灰色）在右侧
- 白色 `.group-body` 圆角 10px，内部 row 之间 1px 淡灰分隔线
- 子板块之间用 1px rgba(0,0,0,0.06) 分隔线

## 14. 对话区快捷操作

### 14.1 三个快捷按钮

输入区上方三个操作按钮，提高对话过程中的知识沉淀效率：

| 按钮 | 作用 | 写入目标 |
|------|------|------|
| 📜 **沉淀到 CLAUDE.md** | 将对话中的通用优化策略提取并追加到全局 CLAUDE.md | `~/Documents/Claude/CLAUDE.md` |
| 📋 **沉淀到本项目 Skills** | 将对话中的项目特定策略提取并追加到当前项目 .md | 当前项目的项目 Skill 文件 |
| 🗜 **压缩上下文** | 压缩当前对话历史为摘要，释放 Token | 替换当前上下文窗口 |

### 14.2 三个按钮的分工

- **沉淀到 CLAUDE.md** vs **沉淀到本项目 Skills**：前者写入全局规则（所有项目生效），后者写入当前项目规则（仅本项目生效）
- **压缩上下文** vs **沉淀**：压缩是清缓存（临时），沉淀是写手册（持久）；两者经常配合——先沉淀再压缩

### 14.3 按钮样式

- 白底灰边框，8px 圆角，1px 阴影
- 11px 字号，500 字重
- 与输入框同行，位于输入区上方
