6. **项目概览打开位置不明确**：是在中央区作为 Tab 打开，还是弹出独立面板？建议：在中央区作为 📋 方案 Tab 的内容展示（因为项目概览就是项目的结构化文档）

## 模块 11：跨模块联动问题

35. **会话切换 → 方案 Tab**：切换到另一个会话后，方案 Tab 应该显示该会话的方案。但方案 Tab 的设计是「AI 自动维护的活文档」——不同会话有不同方案吗？还是整个项目共享一个方案？
    - **建议**：方案 Tab 是**项目级别**的，不随会话切换变化。每个会话的对话内容都会被 AI 综合进入同一个方案文档
36. **会话切换 → 预览 Tab**：预览的文件列表是否切换？应该是——每个会话可能生成不同的文件
37. **会话切换 → 发布 Tab**：发布状态是否切换？发布记录是项目级别的，不应随会话变化
38. **方案 Tab 和项目概览的关系**：方案 Tab 是 AI 自动更新的结构化文档，项目概览是用户手动配置的项目元信息。它们是同一份数据的不同视图？还是独立的两份数据？
    - **建议**：明确为同一份数据。项目概览面板是编辑视图，方案 Tab 是渲染视图
39. **沉淀按钮和方案 Tab 的关系**：点击「沉淀到本项目 Skills」后，方案 Tab 是否自动更新？如果是，如何同步？
40. **沉淀按钮和 CLAUDE.md 的关系**：同上——沉淀后设置面板是否自动刷新？
