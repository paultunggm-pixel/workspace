授权流程 — 登录完全在模型网站进行

App 绝不接触、不可见、不捕获用户在 DeepSeek 网站输入的任何信息

🔄 完整授权链路

1
用户点击「授权登录」
App 调用 ASWebAuthenticationSession， 弹出 系统浏览器 (Safari)，导航到：
https://platform.deepseek.com/oauth/authorize?...
2
用户在 DeepSeek 网站完成登录 🔒 App 完全不可见
输入账号、密码、2FA 验证码——全部发生在 platform.deepseek.com 的页面上。
🚫 App 无法做到的：
  • 看不到登录表单（在 Safari 里，不在 App 内）
  • 无法注入 JavaScript 窃听输入
  • 无法截取 Safari 窗口内容
  • 不知道用户输入了什么
3
DeepSeek 回调 → App 只拿到授权码
DeepSeek 服务器生成一个 临时授权码 (auth code)， 通过回调 URL 传回 App。App 拿这个 code 在后台换 session token。
✅ App 拿到的
auth_code (一次性、5分钟过期)
❌ App 拿不到的
密码、API Key、Cookie
4
Session Token → macOS Keychain
后台用 auth code 换来的 session token，通过 SecItemAdd() 写入系统钥匙串。 之后用量查询从 Keychain 读 token，App 代码中全程无明文。

🛡️ 安全保障总结

🚫
不嵌入 WebView
使用系统 Safari，App 无法注入 JS 或监听输入
🔑
只拿授权码
回调仅返回一次性 auth code，不含任何凭据
🔐
Keychain
Token 入钥匙串，App 不持明文，删登即毁
