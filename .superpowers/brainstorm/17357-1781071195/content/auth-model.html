模型用量获取 — 安全设计 & 授权流程

不存储 API Key · 系统钥匙串托管 · 每小时自动刷新

🔐 安全约束（铁律）

🚫 绝不存储
  • API Key / Secret Key
  • 用户密码
  • 任何纯文本私密信息
  • 本地 plist / UserDefaults 中的 Token
✅ 安全措施
  • 仅存储于 macOS Keychain
  • OAuth 2.0 授权码流
  • Session Token 最小权限
  • 密钥不离开本机

🔄 授权 & 刷新流程

1
首次授权 — OAuth 弹窗
用户点击「🔐 授权登录」→ 打开系统浏览器 → 跳转 DeepSeek Platform 授权页 → 用户确认授权 → 回调 URL 携带 auth code → App 用 code 换取 session token
ⓘ 授权范围仅限：读取用量信息（只读），不可操作账户或密钥
2
Session Token → macOS Keychain
换取到的 session token 通过 SecItemAdd() 存入 macOS 系统钥匙串。 App 本身不持有明文，每次读取通过 Keychain Services API。
🔑 Keychain item: com.claude-code-studio.deepseek-session · access group: 仅本应用
3
每小时自动刷新
后台 Timer.scheduled 每 60 分钟触发 → 从 Keychain 读取 session → 调用 DeepSeek Usage API → 更新 UI 余额卡片。用户也可手动点「🔄」立即刷新。
4
Token 过期 / 登出
Session 过期时显示「⚠️ 授权已过期」→ 用户点击重新授权 → Keychain 旧 token 自动清除。 用户主动登出时，SecItemDelete() 彻底删除 Keychain 记录。
