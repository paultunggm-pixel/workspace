# 千问小讲堂 · 挂载封面生成 — 工作流与需求说明

> 面向：产品 / 设计 / 内容 / 工程  
> 真源副本：与 `~/Desktop/qianwen-cover-generator/封面生成工作流与需求说明.md` 同步维护。

---

## 1. 我们在做什么

把每条课程内容（学名/钩子/插画）批量合成为统一品牌样式的竖版封面（144×192 设计尺寸），并上传 OSS、把封面 URL 回写到内容表。

目前已覆盖的样式：

| 样式 | 适用 | 主文案 | 字体 |
|------|------|--------|------|
| style1 | 小低数学 | 钩子（整段自动换行） | 造字工房圆黑 + 普惠 |
| style2 | 小高数学 | 钩子（整段自动换行） | 普惠 Bold |
| style3 | 语文（单名） | 学名（有《》则仅内文） | HYXinRenWenSong 65W |
| 成语故事 | 成语/寓言 | 学名（大）+ 钩子（小） | 65W 标题 + W 常规钩子（白70%） |

---

## 2. 标准工作流（现状）

```
内容表(Excel/CSV)          设计稿(Figma)            设计成品(可选)
  学名/钩子/url/年级   →     规格(字号/位置/         →   部分篇目老师直接出图
                            字重/换行/配色)
        │                        │                        │
        └──────────┬─────────────┘                        │
                   ▼                                       │
           渲染脚本 render_covers.py  ◄────────────────────┘
        (选样式 → 排版换行 → 自适应毛玻璃徽标 → 文字)
                   ▼
        双版本产出：PNG(无损) + JPG(≤50K 压缩)
                   ▼
            上传 OSS(ossutil) → 得到 CDN URL
                   ▼
        回写内容表：压缩前url(PNG) / 压缩后url(JPG)
```

## 4. 卡点与问题（需多方协作）

| # | 问题 | 建议责任方 |
|---|------|------------|
| 1 | 成品图错名（成语 png 文件名） | 设计 + 审校 |
| 2 | 设计规格未完全 Figma 固化 | 设计 |
| 3 | 浅底白字可读性 | 设计 |
| 4 | 钩子换行孤字 | 内容 + 设计 |
| 5 | 列名 `学名·确认版` / `学名确认版` 不统一 | 产品 + 内容 |
| 6 | 匹配靠中文名非 guid | 全体 |
| 7 | OSS/ossutil 本地硬编码 | 工程 |

---

## 5. 角色分工

- **产品**：输入规范、guid、交付标准  
- **设计**：Figma 规格、成品图命名  
- **内容**：学名/钩子/url 准确；科普学名带类型《篇目》时压图只取篇目  
- **工程**：`run_*.py`、压缩、OSS、回写  
- **审校**：开图核名、可读性抽检  

详细表见桌面工程原文档第四节。
